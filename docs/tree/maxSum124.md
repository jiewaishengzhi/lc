# LeetCode 124 - 二叉树中的最大路径和

## 题目描述

给你一棵二叉树的根节点 `root`，返回其**最大路径和**。

**路径**：从树中任意节点出发，沿父节点-子节点连接，到达任意节点的序列。路径至少包含一个节点，且不需要经过根节点。

```
示例：
        -10
        /  \
       9   20
           / \
          15  7

路径：15→20→7 = 42
答案：42
```

---

## 分析方法

### 核心概念

必须区分两个概念：

| 概念 | 含义 | 能否分叉 |
|------|------|----------|
| **路径和** | 以某节点为分叉点，左+自己+右 | ✅ 可以分叉 |
| **贡献值** | 从某节点向上能传递的最大和 | ❌ 只能选一条路 |

```
      20
     /  \
    15   7

路径和：15 + 20 + 7 = 42（左右都走，可以分叉）
贡献值：20 + max(15, 7) = 35（只能选左边或右边一条路向上）
```

### 算法思路

1. 对每个节点，计算**左右子树的贡献值**（负贡献直接丢弃：`max(0, 贡献)`）
2. 计算以该节点为分叉点的**路径和**：`左贡献 + 节点值 + 右贡献`，更新全局 `maxSum`
3. 返回该节点向上的**贡献值**：`节点值 + max(左贡献, 右贡献)`

### 负数贡献的处理

```
为什么用 max(0, 贡献)？

假设节点值为 3，左子树贡献 -5，右子树贡献 -2：

leftGain  = max(0, -5) = 0  ← 负贡献丢弃，不走左子树
rightGain = max(0, -2) = 0  ← 负贡献丢弃，不走右子树

路径和：3 + 0 + 0 = 3（只取自己，比带上负数子树更好）
贡献值：3 + max(0, 0) = 3

如果不用 max(0, ...)：
路径和：3 + (-5) + (-2) = -4 ← 比单独取3还差！
```

## 代码实现

```java
public class maxSum124 {
    private int maxSum = Integer.MIN_VALUE;

    public int maxPathSum(TreeNode root) {
        dfs(root);
        return maxSum;
    }

    /**
     * 返回以 node 为起点的最大单链贡献值
     * 同时计算以 node 为分叉点的路径和，更新 maxSum
     * @param node 当前节点
     * @return 从当前节点向上的最大贡献值（单链，不分叉）
     */
    private int dfs(TreeNode node) {
        if (node == null) return 0;

        // 递归获取左右子树的贡献值
        // max(0, ...) 表示负贡献不如不要（丢弃负数子树）
        int leftGain = Math.max(0, dfs(node.left));
        int rightGain = Math.max(0, dfs(node.right));

        // 以当前节点为分叉点的路径和（左+自己+右）
        int price = node.val + leftGain + rightGain;
        maxSum = Math.max(maxSum, price);

        // 返回向上贡献的值：自己 + max(左, 右)
        // 只能选一条路，不能分叉
        return node.val + Math.max(leftGain, rightGain);
    }
}
```

                  │
                  ▼
   ┌──────────────────────────────────┐
   │ price = val + leftGain           │
   │         + rightGain              │  ← 以当前节点为分叉点的路径和
   │                                  │
   │ maxSum = max(maxSum, price)      │  ← 更新全局最大值
   └──────────────┬───────────────────┘
                  │
                  ▼
   ┌──────────────────────────────────┐
   │ return val + max(leftGain,       │  ← 向上的单链贡献值
   │                rightGain)        │     只能选一边，不能分叉
   └──────────────────────────────────┘

## 关键点总结

1. **区分路径和与贡献值**：路径和可以分叉（左+根+右），贡献值不能分叉（根+max(左,右)）
2. **max(0, 贡献)**：负贡献直接丢弃，不选该子树比选了更好
3. **maxSum 在递归中维护**：每个节点都可能是分叉点，都要计算路径和
4. **路径不一定要经过根**：所以每个节点独立计算，全局取最大
5. **节点值可以为负**：maxSum 初始化为 `Integer.MIN_VALUE`，不是 0
