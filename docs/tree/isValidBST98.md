# LeetCode 98 - 验证二叉搜索树

## 题目描述

给定一个二叉树的根节点 `root`，判断其是否是一棵合法的**二叉搜索树（BST）**。

**BST 定义**：
- 左子树中**所有节点**的值都**小于**当前节点
- 右子树中**所有节点**的值都**大于**当前节点
- 左右子树也必须分别是 BST

```
示例 1（合法 BST）：         示例 2（非法 BST）：

      5                            5
     / \                          / \
    1   4                        1   4
       / \                          / \
      3   6                        3   6

输出：true                    输出：false
（左<5<右，递归合法）          （节点3在右子树中却<5，不合法）
```

---

## 分析方法

### BST 的核心性质

BST 的**中序遍历**一定是一个**严格递增序列**。基于此性质有两种验证思路：

1. **方法一：递归DFS + 上下界**：每个节点必须在一个合法范围内（lower, upper）
2. **方法二：中序遍历**：判断中序序列是否严格递增

---

## 方法一：递归 DFS + 上下界

### 分析思路

对每个节点维护一个合法区间 `(lower, upper)`：
- 根节点的区间是 `(Long.MIN_VALUE, Long.MAX_VALUE)`
- 进入左子树时，上界更新为当前节点值
- 进入右子树时，下界更新为当前节点值

```
节点值必须满足：lower < node.val < upper
```

### 示例推演

```
输入树：
        5
       / \
      1   4
         / \
        3   6

══════════════════════════════════════════════════════════════
【递归调用过程】
══════════════════════════════════════════════════════════════

初始调用：dfs(节点5, -∞, +∞)

──────────────────────────────────────────────────────────────
【第1层】dfs(节点5, lower=-∞, upper=+∞)
──────────────────────────────────────────────────────────────

检查：5 > -∞ 且 5 < +∞ → ✅ 通过

递归左子树：dfs(节点1, lower=-∞, upper=5)  ← upper更新为5
递归右子树：dfs(节点4, lower=5, upper=+∞)  ← lower更新为5

──────────────────────────────────────────────────────────────
【第2层-左】dfs(节点1, lower=-∞, upper=5)
──────────────────────────────────────────────────────────────

检查：1 > -∞ 且 1 < 5 → ✅ 通过

递归左子树：dfs(null, lower=-∞, upper=1) → return true（空节点）
递归右子树：dfs(null, lower=1, upper=5)  → return true（空节点）

return true ✅

──────────────────────────────────────────────────────────────
【第2层-右】dfs(节点4, lower=5, upper=+∞)
──────────────────────────────────────────────────────────────

检查：4 > 5？ ❌ 不满足！4 ≤ lower(5)

return false ❌

══════════════════════════════════════════════════════════════
【递归回溯过程】
══════════════════════════════════════════════════════════════

dfs(节点5, -∞, +∞)
  ├─ dfs(节点1, -∞, 5)    → true ✅
  │   ├─ dfs(null, -∞, 1) → true ✅
  │   └─ dfs(null, 1, 5)  → true ✅
  │
  └─ dfs(节点4, 5, +∞)    → false ❌  ← 4 ≤ 5
      │
      │ （短路，不再继续递归）
      │
  └─ return false

最终结果：false（不是合法 BST）

══════════════════════════════════════════════════════════════
【为什么节点4不合法？】
══════════════════════════════════════════════════════════════

节点4在根节点5的右子树中，
右子树所有节点都必须 > 5，
但 4 < 5，所以不合法。

        5
       / \
      1   4  ← 在5的右边，但值比5小！非法！
         / \
        3   6
```

### 合法 BST 推演对比

```
输入树：
        5
       / \
      1   7
         / \
        6   8

══════════════════════════════════════════════════════════════

dfs(节点5, -∞, +∞)           → 5 在 (-∞,+∞) ✅
  ├─ dfs(节点1, -∞, 5)       → 1 在 (-∞, 5) ✅
  │   ├─ dfs(null)           → true
  │   └─ dfs(null)           → true
  │   └─ return true ✅
  │
  └─ dfs(节点7, 5, +∞)       → 7 在 (5, +∞) ✅
      ├─ dfs(节点6, 5, 7)    → 6 在 (5, 7) ✅
      │   ├─ dfs(null)       → true
      │   └─ dfs(null)       → true
      │   └─ return true ✅
      │
      └─ dfs(节点8, 7, +∞)   → 8 在 (7, +∞) ✅
          ├─ dfs(null)       → true
          └─ dfs(null)       → true
          └─ return true ✅
      └─ return true ✅

最终结果：true ✅（是合法 BST）

区间传递图：
                5 (-∞, +∞)
               / \
      1(-∞,5)    7(5,+∞)
                 / \
          6(5,7)    8(7,+∞)
```

### 代码实现

```java
public boolean isValidBST(TreeNode root) {
    return dfs(root, Long.MIN_VALUE, Long.MAX_VALUE);
}

private boolean dfs(TreeNode node, long lower, long upper) {
    // 空节点是合法的
    if (node == null) {
        return true;
    }
    // 当前节点值必须严格在 (lower, upper) 范围内
    if (node.val <= lower || node.val >= upper) {
        return false;
    }
    // 递归检查左右子树，更新边界
    return dfs(node.left, lower, node.val)
            && dfs(node.right, node.val, upper);
}
```

```
                          │

                └───────────────────┘
                   │           │
                 Yes          No
                   │           │
                   ▼           ▼
           ┌──────────┐  ┌──────────────────────────┐

                         └──────────────────────────┘
                            │              │
                          Yes             No
                            │              │
                            ▼              ▼
                    ┌──────────┐  ┌──────────────────────┐
                    │return    │  │ 递归左子树            │

                                          ▼

                                          │
                                          ▼

                               └──────────────────────┘
```

## 方法二：中序遍历

### 分析思路

BST 的中序遍历一定是**严格递增序列**。遍历时记录前一个节点的值 `prev`，如果当前节点值 `≤ prev`，则不是 BST。

```
BST 中序遍历：1 → 5 → 6 → 7 → 8（严格递增 ✅）
非BST中序：  1 → 5 → 3 → 4 → 6（3 < 5，非递增 ❌）
```

### 示例推演

```
输入树（非法 BST）：
        5
       / \
      1   4
         / \
        3   6

初始状态：prev = -∞

══════════════════════════════════════════════════════════════
【中序遍历过程】（左 → 根 → 右）
══════════════════════════════════════════════════════════════

inorder(节点5)
  │
  ├── inorder(节点1)               ← 先遍历左子树
  │     │
  │     ├── inorder(null)          ← 1的左子树 → return true
  │     │
  │     ├── 检查：1 > prev(-∞)?    ← 当前节点
  │     │   ✅ 通过
  │     │   prev = 1
  │     │
  │     └── inorder(null)          ← 1的右子树 → return true
  │     └── return true ✅
  │
  ├── 检查：5 > prev(1)?           ← 当前节点
  │   ✅ 通过
  │   prev = 5
  │
  ├── inorder(节点4)               ← 遍历右子树
  │     │
  │     ├── inorder(节点3)         ← 4的左子树
  │     │     │
  │     │     ├── inorder(null)    → return true
  │     │     │
  │     │     ├── 检查：3 > prev(5)?
  │     │     │   ❌ 3 ≤ 5！不满足递增！
  │     │     │
  │     │     └── return false ❌   ← 提前终止
  │     │
  │     └── return false ❌
  │
  └── return false ❌

══════════════════════════════════════════════════════════════
【遍历序列分析】
══════════════════════════════════════════════════════════════

中序遍历顺序：1 → 5 → 3 → ...

期望序列：   严格递增
实际序列：   1, 5, 3 ← 到3时发现 3 < 5，不是递增！

所以返回 false。
```

### 合法 BST 的中序推演对比

```
输入树（合法 BST）：
        5
       / \
      1   7
         / \
        6   8

══════════════════════════════════════════════════════════════

中序遍历顺序：

inorder(5)
  ├── inorder(1)
  │     ├── inorder(null)  → true
  │     ├── 检查：1 > -∞ ✅, prev = 1
  │     └── inorder(null)  → true
  │
  ├── 检查：5 > 1 ✅, prev = 5
  │
  ├── inorder(7)
  │     ├── inorder(6)
  │     │     ├── inorder(null) → true
  │     │     ├── 检查：6 > 5 ✅, prev = 6
  │     │     └── inorder(null) → true
  │     │
  │     ├── 检查：7 > 6 ✅, prev = 7
  │     │
  │     └── inorder(8)
  │           ├── inorder(null) → true
  │           ├── 检查：8 > 7 ✅, prev = 8
  │           └── inorder(null) → true

中序序列：1 → 5 → 6 → 7 → 8（严格递增 ✅）
最终结果：true ✅
```

### 代码实现

```java
private long prev = Long.MIN_VALUE;

public boolean isValidBST2(TreeNode root) {
    return inorder(root);
}

private boolean inorder(TreeNode node) {
    if (node == null) {
        return true;
    }
    // 先遍历左子树
    if (!inorder(node.left)) {
        return false;
    }
    // 检查当前节点是否大于前一个节点
    if (node.val <= prev) {
        return false;
    }
    prev = node.val;
    // 再遍历右子树
    return inorder(node.right);
}
```

## 两种方法对比

| 对比项 | 方法一：上下界 DFS | 方法二：中序遍历 |
|--------|-------------------|-----------------|
| 核心思路 | 维护每个节点的合法区间 | 利用中序递增性质 |
| 参数传递 | lower, upper 边界 | 全局变量 prev |
| 提前终止 | 节点越界立即返回 | prev 逆序立即返回 |
| 空间复杂度 | O(h) 递归栈 | O(h) 递归栈 |
| 代码理解 | 区间收缩较直观 | 中序性质更优雅 |

> h = 树的高度

---

## 复杂度分析

| 复杂度 | 值 | 说明 |
|--------|-----|------|
| 时间 | O(n) | 每个节点最多访问一次 |
| 空间 | O(h) | 递归栈深度，最坏 O(n) |

---

## 关键点总结

1. **用 long 不用 int**：节点的值可能是 `Integer.MIN_VALUE` 或 `Integer.MAX_VALUE`，用 `long` 避免边界问题
2. **严格不等**：BST 要求 `左 < 根 < 右`，不能等于
3. **不能只看左右子节点**：必须保证整个左子树/右子树都满足条件，不是只比较相邻层
