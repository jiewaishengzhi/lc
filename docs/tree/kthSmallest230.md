# LeetCode 230 - 二叉搜索树中第K小的元素

## 题目描述

给定一个二叉搜索树的根节点 `root` 和一个整数 `k`，找出 BST 中第 `k` 小的元素值。

```
示例：
        5
       / \
      3   6
     / \
    2   4
   /
  1

k = 3 → 返回 3

中序序列：1, 2, 3, 4, 5, 6
                   ↑
                第3小
```

---

## 分析方法

BST 的**中序遍历是严格递增序列**，所以中序遍历到第 `k` 个元素就是答案。有两种实现方式：

1. **方法一：递归中序遍历** — 用全局变量 count 计数
2. **方法二：迭代中序遍历** — 用显式栈模拟，可精确提前终止

---

## 方法一：递归中序遍历

### 分析思路

- 中序遍历顺序：左 → 根 → 右
- 每访问一个节点 count++，当 `count == k` 时记录答案
- 找到后 `return` 剪枝，不再继续递归

### 示例推演

```
输入树：
        5
       / \
      3   6
     / \
    2   4
   /
  1

k = 3，count 初始 = 0，result = 0

══════════════════════════════════════════════════════════════
【递归调用过程】
══════════════════════════════════════════════════════════════

inorder(节点5)
  │
  ├── inorder(节点3)
  │     │
  │     ├── inorder(节点2)
  │     │     │
  │     │     ├── inorder(节点1)
  │     │     │     │
  │     │     │     ├── inorder(null) → return
  │     │     │     │
  │     │     │     ├── count++ → count=1
  │     │     │     │   节点值=1，k=3，1≠3，继续
  │     │     │     │
  │     │     │     └── inorder(null) → return
  │     │     │
  │     │     ├── count++ → count=2
  │     │     │   节点值=2，k=3，2≠3，继续
  │     │     │
  │     │     └── inorder(null) → return
  │     │
  │     ├── count++ → count=3
  │     │   节点值=3，k=3，3==3 ✅ 命中！
  │     │   result = 3
  │     │   return ← 剪枝，不再遍历右子树
  │     │
  │     └── （不执行 inorder(节点4)）
  │
  └── （不执行后续遍历）

最终结果：result = 3

══════════════════════════════════════════════════════════════
【中序访问顺序】
══════════════════════════════════════════════════════════════

访问节点：  1    2    [3]
count：     1    2     3   ← 命中，停止
                              后面的 4, 5, 6 不再访问
```

### 代码

```java
// 递归中序遍历
private int count = 0;
private int result = 0;

public int kthSmallest(TreeNode root, int k) {
    inorder(root, k);
    return result;
}

private void inorder(TreeNode node, int k) {
    if (node == null) return;

    inorder(node.left, k);    // 先遍历左子树（更小的值）

    count++;                   // 访问当前节点，计数+1
    if (count == k) {
        result = node.val;     // 第k个就是第k小
        return;                // 剪枝，不再继续
    }

    inorder(node.right, k);   // 再遍历右子树（更大的值）
}
```

### 核心流程图

```
         ┌──────────────┐
         │ inorder(node)│
         └──────┬───────┘
                │
                ▼
        ┌──────────────┐     Yes
        │ node==null?  │────────→ return
        └──────┬───────┘
               │ No
               ▼
     ┌──────────────────┐
     │ inorder(left)    │  ← 左子树（更小的值先处理）
     └────────┬─────────┘
              │
              ▼
     ┌──────────────────┐
     │     count++      │  ← 处理当前节点
     └────────┬─────────┘
              │
              ▼
     ┌──────────────────┐     Yes    ┌──────────────────┐
     │  count == k ?    │──────────→ │ result = node.val│
     └──────┬───────────┘            │ return（剪枝）    │
            │ No                     └──────────────────┘
            ▼
     ┌──────────────────┐
     │ inorder(right)   │  ← 右子树（更大的值后处理）
     └──────────────────┘
```

## 方法二：迭代中序遍历

### 分析思路

用显式栈模拟递归过程：
1. 从当前节点一路向左压栈（模拟递归左子树）
2. 弹出栈顶处理（模拟返回到当前节点）
3. 转向右子树（模拟递归右子树）

**优势**：找到答案后可以立即 `return`，完全终止，比递归剪枝更彻底。

### 示例推演

```
输入树：
        5
       / \
      3   6
     / \
    2   4
   /
  1

k = 3

══════════════════════════════════════════════════════════════
【栈变化过程】
══════════════════════════════════════════════════════════════

──────────────────────────────────────────────────────────────
【轮次1】一路向左压栈
──────────────────────────────────────────────────────────────

从节点5开始，一直走左孩子：
  push(5) → push(3) → push(2) → push(1) → root=null 停止

栈（底→顶）：[5, 3, 2, 1]
root = null
k = 3

──────────────────────────────────────────────────────────────
【轮次2】弹出节点1
──────────────────────────────────────────────────────────────

root = stack.pop() → 节点1
栈：[5, 3, 2]

k-- → k = 2
k≠0，继续

root = 1.right = null

──────────────────────────────────────────────────────────────
【轮次3】弹出节点2
──────────────────────────────────────────────────────────────

内层while：root=null，不压栈

root = stack.pop() → 节点2
栈：[5, 3]

k-- → k = 1
k≠0，继续

root = 2.right = null

──────────────────────────────────────────────────────────────
【轮次4】弹出节点3 ← 命中！
──────────────────────────────────────────────────────────────

内层while：root=null，不压栈

root = stack.pop() → 节点3
栈：[5]

k-- → k = 0 ✅
k == 0，return 节点3的值 = 3

══════════════════════════════════════════════════════════════
【过程汇总】
══════════════════════════════════════════════════════════════

轮次  弹出节点  k值  栈状态         说明
──────────────────────────────────────────────
 -    压栈     3    [5,3,2,1]      一路向左
 2    节点1    2    [5,3,2]        第1小
 3    节点2    1    [5,3]          第2小
 4    节点3    0    [5]            第3小 ✅ return!

后面的节点4、5、6完全没有被访问。
```

### 代码

```java
// 迭代中序遍历
public int kthSmallest2(TreeNode root, int k) {
    // 用 ArrayDeque 模拟递归调用栈
    Deque<TreeNode> stack = new ArrayDeque<>();

    // 循环条件：还有节点没处理 或 栈中还有待回溯的节点
    while (root != null || !stack.isEmpty()) {

        // 一路向左，把路径上所有节点压栈
        // 模拟递归的 inorder(left)，找到最小值
        while (root != null) {
            stack.push(root);
            root = root.left;
        }

        // 弹出栈顶：当前未访问的最小节点
        root = stack.pop();

        // 计数
        k--;
        if (k == 0) {
            return root.val; // 命中，立即返回
        }

        // 转向右子树
        root = root.right;
    }
    return -1; // 不会执行（k一定合法）
}
```

## 两种方法对比

| 对比项 | 递归 | 迭代 |
|--------|------|------|
| 代码量 | 少 | 稍多 |
| 提前终止 | 剪枝后栈帧仍存在 | 精确终止，立即退出 |
| 空间 | O(h) 递归栈 | O(h) 显式栈 |
| 全局变量 | 需要 count + result | 不需要 |
| 多次查询 | 需重置全局变量 | 可复用栈状态 |

---

## 复杂度分析

| 复杂度 | 值 | 说明 |
|--------|-----|------|
| 时间 | O(h + k) | h = 树高（向左压栈），k = 弹栈次数 |
| 空间 | O(h) | 栈深度等于树高度 |

- 平衡树：O(log n + k)
- 链状树（最坏）：O(n)

---

## 关键点总结

1. **BST 中序 = 递增序列**，中序遍历到第 k 个就是第 k 小
2. **迭代法更优**：可以精确控制提前退出，不会浪费后续遍历
3. **k 从1开始**，不是从0开始
4. **剪枝的意义**：只需要遍历前 k 个节点，后面的不需要访问
