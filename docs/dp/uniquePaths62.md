# LeetCode 62. 不同路径 (Unique Paths)

## 一、题目描述

一个机器人位于一个 `m x n` 网格的**左上角**（起始点在下图中标记为 "Start"）。

机器人每次只能**向下**或者**向右**移动一步。机器人试图达到网格的**右下角**（在下图中标记为 "Finish"）。

问**总共有多少条不同的路径**？

### 示例

```
输入：m = 3, n = 7
输出：28

输入：m = 3, n = 2
输出：3
解释：
从左上角开始，总共有 3 条路径可以到达右下角：
1. 向右 -> 向下 -> 向下
2. 向下 -> 向右 -> 向下
3. 向下 -> 向下 -> 向右
```

### 网格示意图

```
     0     1     2     3     4     5     6
   +-----+-----+-----+-----+-----+-----+-----+
 0 |Start|     |     |     |     |     |     |
   +-----+-----+-----+-----+-----+-----+-----+
 1 |     |     |     |     |     |     |     |
   +-----+-----+-----+-----+-----+-----+-----+
 2 |     |     |     |     |     |     |Finish|
   +-----+-----+-----+-----+-----+-----+-----+
```

## 二、分析方法

### 2.1 问题特征识别

1. **计数问题**：要求计算从起点到终点的**路径总数**
2. **移动限制**：只能向右或向下移动（不能回退）
3. **子问题重叠**：到达 (i,j) 的路径数 = 到达 (i-1,j) 的路径数 + 到达 (i,j-1) 的路径数
4. **无后效性**：到达某个格子后，之前是怎么来的不影响后续选择
5. **最优化子结构**：大问题的解可以由小问题的解组合得出

### 2.2 为什么选择动态规划？

| 方法 | 时间复杂度 | 空间复杂度 | 说明 |
|------|-----------|-----------|------|
| 暴力递归 | O(2^(m+n)) | O(m+n) | 大量重复计算 |
| 记忆化递归 | O(m×n) | O(m×n) | 用哈希表存储中间结果 |
| **动态规划** | **O(m×n)** | **O(m×n)** | 迭代填充，效率高 |
| 组合数学 | O(min(m,n)) | O(1) | 数学公式 C(m+n-2, m-1) |

### 2.3 DP 数组定义

定义 `dp[i][j]` 表示：**从起点 (0,0) 到达位置 (i,j) 的不同路径总数**

## 三、状态转移方程推导

### 3.1 核心思路

机器人只能**向右**或**向下**移动，因此：

- 到达 `(i, j)` 的最后一步，只能从 **上方** `(i-1, j)` 或 **左方** `(i, j-1)` 走来
- 所以：`dp[i][j] = dp[i-1][j] + dp[i][j-1]`

### 3.2 特殊情况分析

#### 第一行（i = 0）
- 机器人只能**一直向右**走才能到达
- 每个位置只有 **1 条路径**
- `dp[0][j] = 1`（对于所有 j）

#### 第一列（j = 0）
- 机器人只能**一直向下**走才能到达
- 每个位置只有 **1 条路径**
- `dp[i][0] = 1`（对于所有 i）

#### 起点位置（i = 0, j = 0）
- `dp[0][0] = 1`（起点本身就是 1 种路径）

### 3.3 完整状态转移方程

```
对于任意位置 (i, j)：

         ┌─────────────────────────┐
         │     i == 0 || j == 0    │
         └───────────┬─────────────┘
                /           \
              是             否
              ▼               ▼
        dp[i][j] = 1    dp[i][j] = dp[i-1][j] + dp[i][j-1]
```

### 3.4 转移方向图示

```
                 左方 (i, j-1)
                      │
                      ▼ 向右
              ┌───────────────┐
上方(i-1,j) ──▶│   (i, j)      │
     向下      │  当前位置     │
              └───────────────┘

转移：dp[i][j] = dp[i-1][j] + dp[i][j-1]
       (从上方来)    +    (从左方来)
```

### 3.5 填表顺序

按**从上到下、从左到右**的顺序填充，确保计算 `dp[i][j]` 时，`dp[i-1][j]` 和 `dp[i][j-1]` 已经计算过。

```
填充顺序：
     j=0   j=1   j=2   j=3   ...
   +------+------+-+-+-+-+-+-+
i=0│  1   │  1   │1 │1 │1 │1 │  ← 第一行先填
   +------+------+-+-+-+-+-+-+
i=1│  1   │ (2)  │3 │4 │5 │  │  ← 第二行依次
   +------+------+-+-+-+-+-+-+
i=2│  1   │      │  │  │  │  │
   +------+------+-+-+-+-+-+-+
        ↑
    第一列先填
```

## 五、代码实现

### 5.1 标准动态规划解法

```java
public class uniquePaths62 {
    public int uniquePaths(int m, int n) {
        // dp[i][j] 表示到达 (i,j) 的路径数
        int[][] dp = new int[m][n];

        // 初始化 + 填表
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                // 第一行或第一列：只有一条路径
                if (i == 0 || j == 0) {
                    dp[i][j] = 1;
                }
                // 其他位置：从上方和左方走来
                else {
                    dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
                }
            }
        }

        return dp[m - 1][n - 1];
    }
}
```

### 5.2 代码关键点解析

| 代码行 | 作用 | 说明 |
|--------|------|------|
| `int[][] dp = new int[m][n]` | 创建DP数组 | m行n列 |
| `i == 0 \|\| j == 0` | 边界条件 | 第一行/第一列只有1条路径 |
| `dp[i][j] = dp[i-1][j] + dp[i][j-1]` | 状态转移 | 上方来的 + 左方来的 |

### 5.3 复杂度分析

- **时间复杂度**：O(m × n)，需要填充整个 m×n 的表格
- **空间复杂度**：O(m × n)，需要 m×n 的二维数组

## 七、空间优化版本

由于 `dp[i][j]` 只依赖于 `dp[i-1][j]`（上一行同列）和 `dp[i][j-1]`（当前行前一列），可以用**一维数组**优化：

### 7.1 一维数组优化

```java
public int uniquePathsOptimized(int m, int n) {
    // 只需要一维数组，表示当前行
    int[] dp = new int[n];

    // 初始化第一行
    for (int j = 0; j < n; j++) {
        dp[j] = 1;
    }

    // 从第二行开始
    for (int i = 1; i < m; i++) {
        // dp[0] 永远是1（第一列）
        for (int j = 1; j < n; j++) {
            // dp[j] 原本的值就是 dp[i-1][j]（上一行,还没更新的）
            // dp[j-1] 就是 dp[i][j-1]（当前行左边，已经更新过的）
            dp[j] = dp[j] + dp[j - 1];
        }
    }

    return dp[n - 1];
}
```

### 7.2 优化原理图示

```
二维视角：                    一维视角：
+-----+-----+-----+           dp = [1, 1, 1]
|  1  |  1  |  1  │ ← 第0行
+-----+-----+-----+                 ↓
|  1  |  2  |  3  │ ← 第1行    dp = [1, 2, 3]
+-----+-----+-----+                 ↓
|  1  |  3  |  6  │ ← 第2行    dp = [1, 3, 6]
+-----+-----+-----+

一维更新过程：
dp[j] = dp[j] + dp[j-1]
        ↑           ↑
    上方值      左方值
  (旧dp[j])  (新dp[j-1])
```

**优化后复杂度**：
- 时间复杂度：O(m × n)
- 空间复杂度：O(n)

## 八、数学解法（组合数）

从起点到终点，机器人需要向右走 `n-1` 步，向下走 `m-1` 步，总共 `m+n-2` 步。

问题转化为：在 `m+n-2` 步中选择 `m-1` 步向下（或选择 `n-1` 步向右）。

**组合公式**：
$$C(m+n-2, m-1) = \frac{(m+n-2)!}{(m-1)! \cdot (n-1)!}$$

```java
public int uniquePathsMath(int m, int n) {
    long result = 1;
    int total = m + n - 2;
    int k = Math.min(m - 1, n - 1);  // 选较小的减少计算

    for (int i = 1; i <= k; i++) {
        result = result * (total - k + i) / i;
    }

    return (int) result;
}
```

**数学解法复杂度**：
- 时间复杂度：O(min(m, n))
- 空间复杂度：O(1)

## 十、总结

| 维度 | 内容 |
|------|------|
| **问题类型** | 计数型动态规划（网格路径问题） |
| **DP定义** | `dp[i][j]` = 从起点到 (i,j) 的路径数 |
| **转移方程** | `dp[i][j] = dp[i-1][j] + dp[i][j-1]` |
| **边界条件** | 第一行和第一列都为 1 |
| **填表顺序** | 从上到下，从左到右 |
| **时间复杂度** | O(m × n) |
| **空间复杂度** | O(m × n) 或 O(n) 优化后 |
| **关键点** | 只能向右或向下移动 |
