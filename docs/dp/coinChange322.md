# LeetCode 322 - 零钱兑换（Coin Change）详解

## 题目描述

给定一个整数数组 `coins`，其中 `coins[i]` 表示第 `i` 种硬币的面额；再给定一个整数 `amount`，表示总金额。

要求计算并返回：

- 凑成总金额 `amount` 所需的最少硬币个数
- 如果无论如何都无法凑出该金额，返回 `-1`

题目默认每种硬币都可以使用无限次。

示例：

- 输入：`coins = [1, 2, 5], amount = 11`
- 输出：`3`
- 解释：`11 = 5 + 5 + 1`

---

## 1. 分析方法

这道题本质上是一个 **完全背包 + 最优值** 问题。

### 1.1 为什么适合动态规划

我们要求的是“凑出某个金额时的最少硬币数”，而且：

- 大问题可以拆成小问题
- 凑出金额 `i` 的答案，会依赖更小金额的答案
- 同一个子问题会被多次使用

例如，要求金额 `11` 的最优解时，可能会依赖金额 `10`、`9`、`6` 的最优解；而这些子问题又会继续依赖更小金额的答案。这正是动态规划的典型特征。

### 1.2 为什么贪心不可靠

很多人第一反应是优先选最大面额硬币，但贪心在这题里不一定得到最优解。

反例：

- `coins = [1, 3, 4]`
- `amount = 6`

如果贪心地先拿 `4`，剩下 `2`，只能再拿两个 `1`，一共需要 `3` 枚硬币：

- `4 + 1 + 1`

但真实最优解是：

- `3 + 3`

只需要 `2` 枚硬币。

因此，这题不能单纯依赖贪心，而要系统地比较所有可行转移。

## 2. 状态转移方程的推导

### 2.1 状态定义

定义一维数组 `dp`：

- `dp[i]` 表示“凑出金额 `i` 所需的最少硬币数”

那么最终答案就是：

- `dp[amount]`

### 2.2 最后一步思考法

假设当前正在求 `dp[i]`。

如果最后放入的一枚硬币面额是 `coin`，那么在放入这枚硬币之前，前面的金额必须已经凑成：

- `i - coin`

也就是说：

- 凑成 `i - coin` 的最少硬币数是 `dp[i - coin]`
- 再放入当前这枚 `coin`，总硬币数就变成 `dp[i - coin] + 1`

所以，对于每一种可用硬币 `coin`，都可以尝试这样转移：

- `dp[i] = dp[i - coin] + 1`

因为我们要求“最少”，所以要在所有合法硬币中取最小值：

> `dp[i] = min(dp[i], dp[i - coin] + 1)`，前提是 `i - coin >= 0`

### 2.3 初始化为什么是 `amount + 1`

源码中使用：

```java
int max = amount + 1;
Arrays.fill(dp, max);
```

原因有两个：

1. `amount + 1` 一定比任何合法答案都大
2. 不能用 `Integer.MAX_VALUE`，因为后续执行 `dp[i - coin] + 1` 时可能溢出

为什么 `amount + 1` 一定足够大？

- 如果 `amount` 可以被凑出，最差情况也不过是全用面额 `1` 的硬币
- 那么最多只需要 `amount` 枚硬币
- 所以 `amount + 1` 可以安全作为“不可能/无穷大”的标记

### 2.4 初始条件

- `dp[0] = 0`

含义是：

- 凑出金额 `0` 不需要任何硬币

其余位置初始都设为 `amount + 1`，表示当前暂时还没有找到可行解。

## 3. 详细示例推演


然后设置：

```text

```

```text

```

### 3.2 逐个金额推演

#### 金额 `i = 1`

尝试每一枚硬币：

- `coin = 1`
  - `1 - 1 = 0`，可转移
  - `dp[1] = min(12, dp[0] + 1) = min(12, 1) = 1`
- `coin = 2`
  - `1 - 2 < 0`，不能用
- `coin = 5`
  - `1 - 5 < 0`，不能用

结果：

```text
dp = [0, 1, 12, 12, 12, 12, 12, 12, 12, 12, 12, 12]
```

---

#### 金额 `i = 2`

- `coin = 1`
  - `2 - 1 = 1`
  - `dp[2] = min(12, dp[1] + 1) = min(12, 2) = 2`
- `coin = 2`
  - `2 - 2 = 0`
  - `dp[2] = min(2, dp[0] + 1) = min(2, 1) = 1`
- `coin = 5`
  - `2 - 5 < 0`，不能用

结果：

```text
dp = [0, 1, 1, 12, 12, 12, 12, 12, 12, 12, 12, 12]
```

解释：

- 金额 `2` 最优做法不是 `1 + 1`

### 3.3 无法凑出的情况如何判断

例如：

- `coins = [2]`
- `amount = 3`

初始化：

```text
dp = [0, 4, 4, 4]
```

推演后：

- `dp[1]` 无法更新
- `dp[3]` 也无法更新

最终 `dp[3]` 仍然是初始值 `4`，说明金额 `3` 根本无法被凑出。

所以源码最后使用：

```java
return dp[amount] > amount ? -1 : dp[amount];
```

来区分：

- 能凑出：返回最少硬币数
- 凑不出：返回 `-1`

## 4. 核心代码

下面是与源码一致的核心实现：

```java
import java.util.Arrays;

public class coinChange322 {
    public int coinChange(int[] coins,int amount){
        // 1. 创建 dp 数组
        // 长度为 amount + 1，因为要计算到 dp[amount]
        int[] dp = new int[amount + 1];

        // 2. 初始化
        // 用 amount + 1 作为不可能达到的最大标记，避免 Integer.MAX_VALUE + 1 溢出
        int max = amount + 1;
        Arrays.fill(dp, max);

        // 凑成金额 0 需要 0 个硬币
        dp[0] = 0;

        // 3. 外层循环：枚举每个目标金额
        for(int i = 1; i <= amount; i++){
            // 4. 内层循环：尝试每一种硬币
            for(int coin : coins){
                if(i - coin >= 0){
                    dp[i] = Math.min(dp[i], dp[i - coin] + 1);
                }
            }
        }

        // 5. 检查是否有解
        return dp[amount] > amount ? -1 : dp[amount];
    }
}
```

## 6. 复杂度分析

- 时间复杂度：`O(amount * coins.length)`
- 空间复杂度：`O(amount)`

解释：

- 外层遍历 `1` 到 `amount`
- 内层遍历所有硬币面额
- 只使用了一个长度为 `amount + 1` 的一维数组

---

## 7. 这段代码的关键理解点

这段代码最核心的点有三个：

1. `dp[i]` 不是表示“能不能凑出”，而是表示“最少要几枚硬币”
2. `amount + 1` 是一个很巧妙的无穷大替代值，既安全又避免溢出
3. 每个金额 `i` 都尝试所有硬币，把“最后一枚硬币”的所有可能性比较一遍，最终留下最优解

如果把这三个点真正理解透，这道题就不只是会写代码，而是能彻底明白为什么这样写一定正确。
