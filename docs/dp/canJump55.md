# LeetCode 55. 跳跃游戏 (Jump Game)

## 一、题目描述

给定一个非负整数数组 `nums`，你最初位于数组的**第一个下标**。

数组中的每个元素代表你在该位置可以跳跃的**最大长度**。

判断你是否能够到达数组的**最后一个下标**。

### 示例

```
输入：nums = [2,3,1,1,4]
输出：true
解释：可以先跳 1 步，从下标 0 到达下标 1，然后再从下标 1 跳 3 步到达最后一个下标。

输入：nums = [3,2,1,0,4]
输出：false
解释：无论怎样，总会到达下标为 3 的位置。但该下标的最大跳跃长度是 0，所以永远不可能到达最后一个下标。
```

### 示意图

```
示例1: nums = [2,3,1,1,4]

索引:    0   1   2   3   4
值:     [2] [3] [1] [1] [4]
         ↓
        可以跳1或2步

路径: 0 --跳1步--> 1 --跳3步--> 4 ✓

        ┌───┐     ┌───┐           ┌───┐
        │ 2 │ ──► │ 3 │ ────────► │ 4 │
        └───┘     └───┘           └───┘
         [0]       [1]             [4]
```

```
示例2: nums = [3,2,1,0,4]

索引:    0   1   2   3   4
值:     [3] [2] [1] [0] [4]

从0出发:
  - 跳到1，再跳到2，再跳到3，卡住！(3位置的值是0)
  - 跳到2，再跳到3，卡住！
  - 跳到3，卡住！

无法到达终点 ✗
```

## 二、分析方法

### 2.1 问题特征识别

1. **可达性问题**：判断是否能从起点到达终点
2. **决策问题**：每个位置有多种跳跃选择
3. **无后效性**：到达某个位置后，之前怎么来的不影响后续
4. **最值/覆盖**：关注"最远能到达哪里"

### 2.2 两种解法思路

| 方法 | 思路 | 时间复杂度 | 空间复杂度 |
|------|------|-----------|-----------|
| **贪心算法** | 维护最远可达位置 | O(n) | O(1) |
| 动态规划 | 记录每个位置是否可达 | O(n²) | O(n) |

### 2.3 为什么贪心更优？

- **贪心策略**：我们不需要知道具体怎么跳，只需要知道"最远能跳到哪里"
- **关键洞察**：如果能到达位置 `i`，那么 `[0, i]` 范围内的所有位置都能到达
- **更新规则**：遍历过程中，不断更新能到达的最远位置

## 三、方法一：贪心算法（当前代码）

### 3.1 核心思想

维护一个变量 `maxReach`，表示**当前能到达的最远位置**。

- 遍历数组每个位置 `i`
- 如果 `i > maxReach`，说明当前位置不可达，返回 `false`
- 否则，更新 `maxReach = max(maxReach, i + nums[i])`
- 如果 `maxReach >= n-1`，说明可以到达终点

### 3.2 状态定义

```
maxReach = 当前能到达的最远索引位置
```

### 3.3 贪心策略证明

**为什么只需要记录最远位置？**

假设我们能到达位置 `i`，那么在 `i` 处可以跳跃 `[1, nums[i]]` 步。

如果我们选择跳 `k` 步（`1 ≤ k ≤ nums[i]`），到达 `i+k`。

由于我们想要最大化覆盖范围，贪心地选择**更新最远可达位置**是正确的，因为：
- 能到达更远的位置，意味着中间的所有位置也都能到达
- 我们不需要具体记录"怎么跳"，只需要知道"能跳多远"

## 五、代码实现

### 5.1 贪心算法（当前代码）

```java
public class canJump55 {
    public boolean canJump(int[] nums) {
        // maxReach 表示当前能跳到的最远位置
        int maxReach = 0;
        int n = nums.length;

        for (int i = 0; i < n; i++) {
            // 1. 如果当前位置 i 已经超过了最远能到达的位置
            //    说明中间断了，过不来，失败
            if (i > maxReach) {
                return false;
            }

            // 2. 更新最远覆盖范围
            //    在位置 i，能跳 nums[i] 步，所以最远能到 i + nums[i]
            //    取之前的 maxReach 和现在算出来的最大值
            maxReach = Math.max(maxReach, i + nums[i]);

            // 3. 剪枝优化：如果覆盖范围已经到了终点，提前结束
            if (maxReach >= n - 1) {
                return true;
            }
        }
        return false;
    }
}
```

### 5.2 代码关键点解析

| 代码行 | 作用 | 说明 |
|--------|------|------|
| `maxReach = 0` | 初始化 | 起点是0，初始能到达位置0 |
| `i > maxReach` | 可达性检查 | 如果当前位置不可达，直接返回false |
| `maxReach = Math.max(...)` | 更新最远位置 | 贪心地扩展覆盖范围 |
| `maxReach >= n-1` | 剪枝 | 提前终止，提高效率 |

### 5.3 复杂度分析

- **时间复杂度**：O(n)，只需一次遍历
- **空间复杂度**：O(1)，只用了常数变量

## 七、方法二：动态规划（对比学习）

虽然贪心是最优解，但理解 DP 方法有助于加深对问题的理解。

### 7.1 DP 状态定义

```
dp[i] = true 表示位置 i 可以到达
dp[i] = false 表示位置 i 不可达
```

### 7.2 状态转移

```
dp[0] = true (起点)

对于每个位置 i (从1到n-1):
    对于每个位置 j (从0到i-1):
        如果 dp[j] = true 且 j + nums[j] >= i:
            dp[i] = true
            break
```

### 7.3 DP 代码实现

```java
public boolean canJumpDP(int[] nums) {
    int n = nums.length;
    boolean[] dp = new boolean[n];
    dp[0] = true;  // 起点可达

    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++) {
            // 如果 j 可达，且从 j 能跳到 i
            if (dp[j] && j + nums[j] >= i) {
                dp[i] = true;
                break;
            }
        }
    }

    return dp[n - 1];
}
```

### 7.4 DP 示例推演：nums = [2,3,1,1,4]

```
初始化: dp = [true, false, false, false, false]
                       [T,    F,     F,     F,     F]

i=1: 检查能否从0跳到1
     dp[0]=true 且 0+nums[0]=2 >= 1 ✓
     dp[1] = true
     dp = [T, T, F, F, F]

i=2: 检查能否从0或1跳到2
     dp[0]=true 且 0+nums[0]=2 >= 2 ✓
     dp[2] = true
     dp = [T, T, T, F, F]

i=3: 检查能否从0,1,2跳到3
     dp[1]=true 且 1+nums[1]=4 >= 3 ✓
     dp[3] = true
     dp = [T, T, T, T, F]

i=4: 检查能否从0,1,2,3跳到4
     dp[1]=true 且 1+nums[1]=4 >= 4 ✓
     dp[4] = true
     dp = [T, T, T, T, T]

return dp[4] = true
```

### 7.5 方法对比

| 维度 | 贪心 | 动态规划 |
|------|------|----------|
| 时间复杂度 | O(n) | O(n²) |
| 空间复杂度 | O(1) | O(n) |
| 思路 | 维护最远可达 | 记录每个位置是否可达 |
| 适用性 | 本题最优 | 更通用的思路 |

## 八、总结

| 维度 | 内容 |
|------|------|
| **问题类型** | 可达性判断（贪心/DP） |
| **贪心核心** | 维护 `maxReach`（最远可达位置） |
| **关键条件** | 如果 `i > maxReach`，则不可达 |
| **更新规则** | `maxReach = max(maxReach, i + nums[i])` |
| **时间复杂度** | O(n) |
| **空间复杂度** | O(1) |
| **贪心正确性** | 最远可达位置包含了所有中间位置的可达性 |
