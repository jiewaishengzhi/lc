# LeetCode 91. 解码方法 (Decode Ways)

## 一、题目描述

一条包含字母 `A-Z` 的消息通过以下映射进行了编码：

```
'A' -> "1"
'B' -> "2"
...
'Z' -> "26"
```

要解码一个已编码的消息，所有数字必须反向映射回字母（可能有多种方法）。

给定一个只包含数字的**非空**字符串 `s`，请计算并返回**解码方法的总数**。

### 示例

```
输入：s = "12"
输出：2
解释：可以解码为 "AB"（1 2）或 "L"（12）

输入：s = "226"
输出：3
解释：可以解码为 "BZ"（2 26）、"VF"（22 6）或 "BBF"（2 2 6）

输入：s = "06"
输出：0
解释："06" 无法映射到 'F'，因为 "6" 和 "06" 不同
```

---

## 二、分析方法

### 2.1 问题特征识别

1. **最值/计数问题**：要求计算解码方法的**总数**，这是典型的计数型动态规划问题
2. **子问题重叠**：前 `i` 个字符的解码方法数，依赖于前 `i-1` 和前 `i-2` 个字符的解码方法数
3. **无后效性**：当前状态一旦确定，之后的决策不会受到之前如何到达该状态的影响
4. **最优化子结构**：大问题的解可以由小问题的解推导得出

### 2.2 为什么选择动态规划？

- **暴力解法的问题**：如果用递归枚举所有可能的分割方式，时间复杂度为 O(2^n)
- **重复计算**：很多子问题会被反复计算，例如 "226" 中，计算 "2|26" 和 "22|6" 时都会计算 "6" 的解码方式
- **DP 优势**：通过记录中间结果，将时间复杂度降为 O(n)

### 2.3 DP 数组定义

定义 `dp[i]` 表示：**前 `i` 个字符（即 `s[0..i-1]`）的解码方法总数**

> 注意：`dp` 数组长度为 `n+1`，使用 1-based 索引，方便处理边界情况

## 三、状态转移方程推导

### 3.1 核心思路

对于当前位置 `i` 的字符 `s[i-1]`，有两种解码选择：

#### 情况 A：单独解码

- **条件**：当前字符不是 `'0'`（即字符为 '1'-'9'）
- **含义**：当前字符可以单独解码为一个字母 A-I
- **转移**：`dp[i] += dp[i-1]`
- **解释**：如果前 `i-1` 个字符有 `dp[i-1]` 种解码方式，那么当前字符单独解码后，总数就是 `dp[i-1]` 种

#### 情况 B：与前一位组合解码

- **条件**：
  - 存在前一位字符（`i > 1`）
  - 组合数字在 `[10, 26]` 范围内（对应字母 J-Z）
- **含义**：当前字符和前一个字符组合成一个两位数，解码为字母 J-Z
- **转移**：`dp[i] += dp[i-2]`
- **解释**：如果前 `i-2` 个字符有 `dp[i-2]` 种解码方式，那么最后两位组合解码后，总数就是 `dp[i-2]` 种

### 3.2 完整状态转移方程

```
dp[i] = 0  // 初始化为0

// 情况A：单独解码
if (s[i-1] != '0'):
    dp[i] += dp[i-1]

// 情况B：组合解码
if (i > 1 && 10 <= (s[i-2]-'0')*10 + (s[i-1]-'0') <= 26):
    dp[i] += dp[i-2]
```

### 3.3 边界条件

- `dp[0] = 1`：空字符串有 1 种解码方式（这是为了让 `dp[2]` 能正确累加 `dp[0]`）
- `dp[1]`：取决于第一个字符是否为 '0'

### 3.4 状态转移图示

```
位置:     0    1    2    3    ...    n
字符:     -   s[0] s[1] s[2]  ...  s[n-1]
dp值:   dp[0] dp[1] dp[2] dp[3] ...  dp[n]

转移关系:
                    单独解码
           dp[i-1] --------> dp[i]

                    组合解码
           dp[i-2] --------> dp[i]
```

## 四、示例推演

### 4.1 示例 1：s = "226"

**初始化**：
```
n = 3
dp = [1, 0, 0, 0]  // dp[0]=1, 其余待计算
s = "226"
索引:  0 1 2
```

**第1轮：i=1, curr='2'**

```
curr = s[0] = '2'

情况A：'2' != '0' ✓
  dp[1] += dp[0] = 0 + 1 = 1

情况B：i=1, 不满足 i>1，跳过

dp[1] = 1
含义：前1个字符"2"只有1种解码方式 -> "B"
```

**第2轮：i=2, curr='2'**

```
curr = s[1] = '2'

情况A：'2' != '0' ✓
  dp[2] += dp[1] = 0 + 1 = 1

情况B：i=2 > 1 ✓
  prev = s[0] = '2'
  val = 2*10 + 2 = 22
  10 <= 22 <= 26 ✓
  dp[2] += dp[0] = 1 + 1 = 2

dp[2] = 2
含义：前2个字符"22"有2种解码方式
      方式1: "2|2" -> "BB"
      方式2: "22" -> "V"
```

**第3轮：i=3, curr='6'**

```
curr = s[2] = '6'

情况A：'6' != '0' ✓
  dp[3] += dp[2] = 0 + 2 = 2

情况B：i=3 > 1 ✓
  prev = s[1] = '2'
  val = 2*10 + 6 = 26
  10 <= 26 <= 26 ✓
  dp[3] += dp[1] = 2 + 1 = 3

dp[3] = 3
含义：前3个字符"226"有3种解码方式
      方式1: "2|2|6" -> "BBF"
      方式2: "22|6" -> "VF"
      方式3: "2|26" -> "BZ"
```

**最终结果**：
```
dp = [1, 1, 2, 3]
return dp[3] = 3
```

### 4.2 示例 2：s = "06"

**初始化**：
```
n = 2
dp = [1, 0, 0]
s = "06"
```

**第1轮：i=1, curr='0'**

```
情况A：'0' == '0' ✗ 不满足，跳过
情况B：i=1, 不满足 i>1，跳过

dp[1] = 0  // 无法解码以0开头的字符串
```

**第2轮：i=2, curr='6'**

```
情况A：'6' != '0' ✓
  dp[2] += dp[1] = 0 + 0 = 0

情况B：i=2 > 1 ✓
  prev = s[0] = '0'
  val = 0*10 + 6 = 6
  10 <= 6 <= 26 ✗ 不满足（06不是有效的两位数编码）

dp[2] = 0
```

**最终结果**：
```
dp = [1, 0, 0]
return dp[2] = 0
```

### 4.3 示例 3：s = "10"

**初始化**：
```
n = 2
dp = [1, 0, 0]
s = "10"
```

**第1轮：i=1, curr='1'**

```
情况A：'1' != '0' ✓
  dp[1] += dp[0] = 1

情况B：i=1, 跳过

dp[1] = 1
```

**第2轮：i=2, curr='0'**

```
情况A：'0' == '0' ✗ 跳过

情况B：i=2 > 1 ✓
  prev = s[0] = '1'
  val = 1*10 + 0 = 10
  10 <= 10 <= 26 ✓
  dp[2] += dp[0] = 0 + 1 = 1

dp[2] = 1
含义："10"只能解码为"J"，"1|0"中0单独无效
```

**最终结果**：
```
dp = [1, 1, 1]
return dp[2] = 1
```

## 五、代码实现

```java
public class numDecondings91 {
    public int numDecoding(String s) {
        int n = s.length();
        // dp[i] 表示前i个字符的解码方法总数
        int[] dp = new int[n + 1];

        // 1. 初始化
        dp[0] = 1; // 空串算一种

        // 2. 遍历字符串 i从1开始 对应s中的第i个字符 s[i-1]
        for (int i = 1; i <= n; i++) {
            // 获取当前字符
            char curr = s.charAt(i - 1);

            // --- 情况 A：单独解码当前位 ---
            // 只要不是 '0'，就可以单独作为一个字母 A-I
            if (curr != '0') {
                dp[i] += dp[i - 1];
            }

            // --- 情况 B：和前一位组合解码 ---
            // 必须要有前一位存在 (i > 1)
            if (i > 1) {
                char prev = s.charAt(i - 2);
                // 组合数字val
                int val = (prev - '0') * 10 + (curr - '0');

                // 只有在 10 到 26 之间才是合法的字母 J-Z
                // 注意：像 "06" 这种，val是6，不在10-26范围内，会被自动过滤
                if (val >= 10 && val <= 26) {
                    dp[i] += dp[i - 2];
                }
            }
        }
        return dp[n];
    }
}
```

### 5.1 代码关键点解析

| 代码行 | 作用 | 说明 |
|--------|------|------|
| `dp[0] = 1` | 初始化空串 | 让 `dp[2]` 能正确累加 |
| `dp[i] += dp[i-1]` | 单独解码 | 当前字符为1-9时可单独解码 |
| `i > 1` | 边界检查 | 确保有前一个字符存在 |
| `val >= 10 && val <= 26` | 组合有效性 | 过滤 "07" 等无效组合 |

### 5.2 复杂度分析

- **时间复杂度**：O(n)，只需一次遍历
- **空间复杂度**：O(n)，使用 dp 数组

## 七、空间优化版本（可选）

由于 `dp[i]` 只依赖于 `dp[i-1]` 和 `dp[i-2]`，可以用三个变量代替数组：

```java
public int numDecodingOptimized(String s) {
    int n = s.length();
    int prev2 = 1;  // dp[i-2]
    int prev1 = 1;  // dp[i-1]
    int curr = 0;   // dp[i]

    for (int i = 1; i <= n; i++) {
        curr = 0;

        // 情况A：单独解码
        if (s.charAt(i - 1) != '0') {
            curr += prev1;
        }

        // 情况B：组合解码
        if (i > 1) {
            int val = (s.charAt(i - 2) - '0') * 10 + (s.charAt(i - 1) - '0');
            if (val >= 10 && val <= 26) {
                curr += prev2;
            }
        }

        // 滚动更新
        prev2 = prev1;
        prev1 = curr;
    }

    return curr;
}
```

**优化后复杂度**：
- 时间复杂度：O(n)
- 空间复杂度：O(1)

## 八、总结

| 维度 | 内容 |
|------|------|
| **问题类型** | 计数型动态规划 |
| **DP定义** | `dp[i]` = 前 i 个字符的解码方法数 |
| **转移方程** | 单独解码 + 组合解码（需满足条件） |
| **时间复杂度** | O(n) |
| **空间复杂度** | O(n) 或 O(1) 优化后 |
| **关键点** | '0' 的特殊处理，组合数范围 [10, 26] |
